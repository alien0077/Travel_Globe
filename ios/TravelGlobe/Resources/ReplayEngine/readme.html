快速開始

1. 打開回放

進入首頁後會自動載入範例航班 CI100，也可以在右側「… / 更多」打開「航班預載 / API key」只輸入航班號。若有 aviationstack API key，可填入第一列「aviationstack API key（保存在本機）」；查詢成功後會快取航班，之後 API 失敗時會先用歷史航班回補。

2. 播放或暫停

按 Play 或 Pause 控制飛行回放。時間軸可拖曳到任意進度。

3. 切換視角

預設是 Global View。你可以切到跟機、座艙、窗邊、尾追、俯視或 Orbit cinema。

4. 匯出紀錄

按 Export 下載 Travel Globe 專用包；按 GPX 或 KML 給 GPS 工具或 Google Earth 使用。

畫面說明

HUD 卡片 顯示航班號、航線、高度、速度、地速、航向、已飛距離、總距離與 ETA。
Flight Overlay 白線代表 Flight Plan，綠線代表目前已飛航跡，黃色點代表 Takeoff、TOC、TOD、Landing 等事件。
旅遊紀錄 可點擊旅程事件直接跳到該時間點，中央紀錄卡會同步顯示地點、區域與航段資訊。
航班預載 / API key 手機版按右側「… / 更多」會直接展開此區。可輸入 aviationstack API key、航班號、起降機場、日期、時間與機型。流程會先嘗試 aviationstack，再使用本機航班快取，最後 fallback 到離線 seed 或手動 IATA。
Travel Atlas 顯示旅程統計、起降機場 detail、OurAirports frequency/navaid 摘要、離線包、本機歷史旅程與通知狀態。
Below / Nearby 顯示目前下方區域、正在穿越的空域/海域，以及附近城市或地標距離。

操作控制

  • Play / Pause：開始或暫停回放。
  • 1x / 5x / 20x / 100x：調整回放速度。
  • Camera：切換視角。
  • Import：匯入 .json 或 .travelglobe 旅程檔。
  • Export：匯出完整 Travel Globe 回放包。
  • Share：匯出隱私處理後的分享版 JSON。
  • Journal：產生離線飛行日誌 Markdown。
  • Pack：快速安裝 Core Global Atlas；完整離線包安裝/刪除可在 Travel Atlas 面板操作。

Travel Atlas

  • 機場 detail：航線起點與終點會顯示機場名稱、城市、跑道數、常用頻率與附近導航台。這些資料來自本機 OurAirports aviation context index。
  • 離線資料包：Core Global Atlas 與 East Asia Flight Context 可在面板內安裝或刪除。狀態會保存在本機瀏覽器/iOS WebView localStorage，不會上傳。
  • 本機歷史旅程：已載入或建立過的 journey 會出現在本機歷史清單，可從 Travel Atlas 載入或刪除。刪除只移除本機儲存，不會影響已匯出的檔案。
  • 通知：Replay Engine 會把目前 notification rules 送到 iOS native local notification；瀏覽器模式只會顯示提醒狀態。

視角模式

  • Global View：像機上娛樂系統的地球總覽，適合看完整航線。
  • Follow camera：鏡頭跟著飛機移動，適合觀察實際航跡。
  • Orbit cinema：鏡頭繞著飛機旋轉，適合影片式回放。
  • Cockpit view：接近座艙視角，朝飛行方向看。
  • Left / Right window：模擬窗邊視角，可看地球與航跡。
  • Tail chase：從機尾後方追蹤飛機。
  • Top down：從上方看航線，適合判斷偏航與航跡形狀。

匯出資料

  • .travelglobe：App 自己的完整回放格式，保留旅程、航線、事件與設定。
  • .json：適合開發、檢查 schema 或分享給其他工具處理。
  • .gpx：可匯入 GPS 工具、運動紀錄工具或航跡分析工具。
  • .kml：可匯入 Google Earth，會包含白色 Flight Plan、綠色 Actual Track 與事件點。

離線使用

Replay Engine 目前的核心回放不依賴外部網路；Blue Marble、Earth lights/cloud/specular 地球貼圖、Natural Earth 海岸線/國界/城市/地理區域、OurAirports 機場/頻率/導航台索引、航線、授權飛機模型庫和範例資料都包在本地 build 裡。飛行前可先打開網頁或 iOS app 確認載入，飛行中即使沒有網路仍可回放已內建或已匯入的旅程。

「航班預載」會優先使用使用者本機保存的 aviationstack API key 查詢航班；成功後會寫入本機航班快取。若 API 失敗、網路不可用或沒有 key，會依序使用本機歷史航班、離線班表 seed 與手動 IATA 欄位建立 Great Circle 預估 route、事件、ETA 和旅遊紀錄。

Live GPS 飛行模式

iOS app 可用 iPhone CoreLocation 接收真實 GPS，透過 native bridge 即時送進 Replay Engine。此模式不需要行動網路；飛航模式下 GPS 仍可能可用，但定位品質會受座位、窗邊距離、機身遮蔽與首次定位時間影響。

  1. 起飛前先開 Travel Globe，按 Open Replay Engine，手機版再按右側「… / 更多」，在「航班預載 / API key」輸入 aviationstack API key（可留空）、航班、起飛/抵達、日期、時間與機型，按 套用航線。可重複套用多個 flight plan；此時只建立待飛行航線，不會建立完成的旅遊紀錄。
  2. 回到 iOS 主畫面，在 Record Into 選單挑要錄入的行程，再按 Start。iOS 會把 GPS recording 綁到被選到航線的 segmentId；若沒有套用航線，則建立 GPS-only recording。
  3. 按 Open Replay Engine 回到地球畫面。收到第一筆 GPS 後，控制列會進入 LIVE，HUD 會顯示 Live GPS 或 GPS 狀態。
  4. 若需要開飛航模式，開啟後把手機放在靠窗位置，等待 GPS 重新定位。定位恢復後飛機會繼續沿真實位置移動。
  5. 短暫斷訊時，畫面會用最後可信速度與航向最多推算 90 秒，HUD 會顯示 GPS 推算；超過 90 秒會停止外推，飛機停在最後可信位置並顯示 GPS signal lost。
  6. 落地或不需要紀錄時回到 iOS 主畫面按 Stop，旅程會保存成真實 GPS 航跡，並把 completed journey 回寫到 Replay Engine 的旅遊紀錄與 Travel Atlas。

資料庫只保存品質合格且有意義變化的真實 GPS 點；畫面推算點只留在記憶體，不會寫入 SQLite，也不會污染 GPX/KML 或之後的飛行紀錄。飛行 profile 會丟棄座標無效、水平精度過差或時間倒退的點，並以至少 5 秒、超過 250 公尺、航向或高度明顯變化作為保存門檻。

如果只是想在某個小鎮、車站、景點留下「到此一遊」，不用開始整段 GPS recording。回到 iOS 主畫面按 GPS打卡 會抓一次目前 CoreLocation 並新增到目前行程；按 照片打卡 會從該行程時間範圍內含 GPS 的照片匯入打卡點。兩者都寫入 visit_points，不會改動原始飛行 GPS 航跡。

旅遊紀錄流程

  1. 套用航線：只建立 planned flight plan，包含航班號、起降機場、機型、預估航線與 segmentId。可先建立多筆；這一步不算完成旅遊紀錄。
  2. 選擇行程並開始 GPS：按 iOS 主畫面的 Record Into 選單挑目標 flight plan，再按 Start。native recording 會使用被選到的 flight plan；每個真實 GPS 點都會保存同一個 segmentId，用來對回該航段。
  3. 停止 GPS：按 Stop 後，app 才把 recording 標記為 completed，產生實際 GPS 航跡、起訖事件、旅遊紀錄與 Travel Atlas 摘要。
  4. GPS打卡 / 照片打卡：在 iOS 主畫面可把目前 GPS 或照片 EXIF GPS 加到目前行程。Replay Engine 收到 native bridge 後會顯示成「GPS打卡」或「照片打卡」旅遊紀錄。
  5. 人工編輯：在「旅遊紀錄」面板可新增事件、修改目前紀錄、調整分類/時間、附加本機照片縮圖、復原上次編輯、隱藏紀錄，或編輯航線摘要。人工修改只寫入 edit overlay、media metadata 或 journey metadata，不會改掉原始 GPS 點。
  6. 資料對應：GPS record 是原始軌跡；Visit Point 是同一行程下的到此一遊點；Travel Record 是同一個 segmentId 下整理出的旅遊事件。刪除/隱藏旅遊事件不會刪除 SQLite 裡的真實 GPS。

iOS App 使用

  • Start Flight Recording：開始紀錄 GPS 航跡。
  • Stop Recording：停止並保存旅程。
  • Open Replay Engine：打開內嵌 Web Replay Engine。
  • GPS打卡：用目前 iPhone GPS 在目前行程新增一筆 visit point。
  • 照片打卡：從行程時間範圍內有 GPS 的 PhotoKit 照片匯入 visit points。
  • 載入最新紀錄：把 SQLite 最新 journey queue 回 Replay Engine，適合 app 重新打開後繼續看剛錄完的旅程。
  • Photos：要求或更新 PhotoKit 權限，用於照片 GPS 匯入。
  • Notifications：要求或更新通知權限；Replay Engine notification rules 會透過 native bridge 排程 local notification。

實際飛行時的建議順序是：先在 Replay Engine 套用航線，再回 iOS 主畫面選 Record Into 並按 Start，最後回 Replay Engine 看 live 地球；飛行中不要用瀏覽器版本紀錄，因為 live GPS bridge 只存在 iOS app 內嵌 WKWebView。

測試建議

  1. 先用範例 CI100 檢查地球、白線、綠線、A380、HUD 和 timeline 是否正常。
  2. 拖動時間軸，確認綠色已飛航跡會跟著變長，ETA 和距離會更新。
  3. 切換每個 Camera 模式，確認畫面不是黑屏且可以滑動旋轉、縮放。
  4. 按 GPX、KML 匯出後，用文字編輯器確認有 track point 和 waypoint。
  5. 手機版會把地球、Travel Atlas、航班預載、旅遊紀錄與控制列排成可捲動版面。
  6. 實機測 live GPS 時，可先在地面走動或開車短距離測試：HUD 應出現 LIVE，綠色已飛航跡應只跟著真實 GPS 點延伸。
