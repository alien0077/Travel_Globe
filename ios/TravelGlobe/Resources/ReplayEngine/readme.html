快速開始

1. 打開回放

進入首頁後會自動載入範例航班 CI100，也可以在「航班預載」只輸入航班號，讓離線班表先解析起降機場並建立回放旅程。

2. 播放或暫停

按 Play 或 Pause 控制飛行回放。時間軸可拖曳到任意進度。

3. 切換視角

預設是 Global View。你可以切到跟機、座艙、窗邊、尾追、俯視或 Orbit cinema。

4. 匯出紀錄

按 Export 下載 Travel Globe 專用包；按 GPX 或 KML 給 GPS 工具或 Google Earth 使用。

畫面說明

HUD 卡片 顯示航班號、航線、高度、速度、地速、航向、已飛距離、總距離與 ETA。
Flight Overlay 白線代表 Flight Plan，綠線代表目前已飛航跡，黃色點代表 Takeoff、TOC、TOD、Landing 等事件。
旅遊紀錄 可點擊旅程事件直接跳到該時間點，中央紀錄卡會同步顯示地點、區域與航段資訊。
航班預載 使用離線班表與機場索引建立預估 Flight Plan。已知航班如 CI100 可自動解析 TPE → NRT，手動起降機場欄位可留空或覆寫。
Below / Nearby 顯示目前下方區域、正在穿越的空域/海域，以及附近城市或地標距離。

操作控制

  • Play / Pause：開始或暫停回放。
  • 1x / 5x / 20x / 100x：調整回放速度。
  • Camera：切換視角。
  • Import：匯入 .json 或 .travelglobe 旅程檔。
  • Export：匯出完整 Travel Globe 回放包。
  • Share：匯出隱私處理後的分享版 JSON。
  • Journal：產生離線飛行日誌 Markdown。
  • Pack：安裝示範離線資料包。

視角模式

  • Global View：像機上娛樂系統的地球總覽，適合看完整航線。
  • Follow camera：鏡頭跟著飛機移動，適合觀察實際航跡。
  • Orbit cinema：鏡頭繞著飛機旋轉，適合影片式回放。
  • Cockpit view：接近座艙視角，朝飛行方向看。
  • Left / Right window：模擬窗邊視角，可看地球與航跡。
  • Tail chase：從機尾後方追蹤飛機。
  • Top down：從上方看航線，適合判斷偏航與航跡形狀。

匯出資料

  • .travelglobe：App 自己的完整回放格式，保留旅程、航線、事件與設定。
  • .json：適合開發、檢查 schema 或分享給其他工具處理。
  • .gpx：可匯入 GPS 工具、運動紀錄工具或航跡分析工具。
  • .kml：可匯入 Google Earth，會包含白色 Flight Plan、綠色 Actual Track 與事件點。

離線使用

Replay Engine 目前的核心回放不依賴外部網路；Blue Marble 地球貼圖、Natural Earth 海岸線/國界、OurAirports 機場/頻率/導航台索引、航線、城市光點、授權飛機模型庫和範例資料都包在本地 build 裡。飛行前可先打開網頁或 iOS app 確認載入，飛行中即使沒有網路仍可回放已內建或已匯入的旅程。

「航班預載」目前使用離線機場索引與大圓航線推估 route、事件、ETA 和旅遊紀錄；實際航路、航班狀態與延誤資訊可在後續接第三方航班 API 後覆蓋同一個 provider 流程。

Live GPS 飛行模式

iOS app 可用 iPhone CoreLocation 接收真實 GPS，透過 native bridge 即時送進 Replay Engine。此模式不需要行動網路；飛航模式下 GPS 仍可能可用，但定位品質會受座位、窗邊距離、機身遮蔽與首次定位時間影響。

  1. 起飛前先開 Travel Globe，按 Start Flight Recording，允許定位權限；長航班建議保持電量充足。
  2. 按 Open Replay Engine 進入地球畫面。收到第一筆 GPS 後，控制列會進入 LIVE，HUD 會顯示 Live GPS 或 GPS 狀態。
  3. 若需要開飛航模式，開啟後把手機放在靠窗位置，等待 GPS 重新定位。定位恢復後飛機會繼續沿真實位置移動。
  4. 短暫斷訊時，畫面會用最後可信速度與航向最多推算 90 秒，HUD 會顯示 GPS 推算；超過 90 秒會停止外推，飛機停在最後可信位置並顯示 GPS signal lost。
  5. 落地或不需要紀錄時回到 iOS 主畫面按 Stop Recording，旅程會保存成真實 GPS 航跡。

資料庫只保存品質合格且有意義變化的真實 GPS 點；畫面推算點只留在記憶體，不會寫入 SQLite，也不會污染 GPX/KML 或之後的飛行紀錄。飛行 profile 會丟棄座標無效、水平精度過差或時間倒退的點，並以至少 5 秒、超過 250 公尺、航向或高度明顯變化作為保存門檻。

資料來源

  • Airport and runway data provided by OurAirports. 頻率與導航台資料也來自 OurAirports 離線 CSV。
  • Made with Natural Earth. Natural Earth 提供 110m 國界與海岸線資料基礎。
  • NASA Visible Earth / Blue Marble 提供地球影像素材；Travel Globe 不代表 NASA 背書。
  • 飛機模型庫只接受 CC0 或 CC BY、可商業使用、可修改、非 Editorial 的 GLB/glTF 模型；若模型來自 Sketchfab，App 會保留作者、授權、模型連結與 Sketchfab 來源標示，並可套用 Travel Globe 中性塗裝。
  • 目前航班預載第一版使用離線班表種子與 Great Circle 預估航線，尚未接入付費真實航班 API。

iOS App 使用

  • Start Flight Recording：開始紀錄 GPS 航跡。
  • Stop Recording：停止並保存旅程。
  • Open Replay Engine：打開內嵌 Web Replay Engine。
  • Request Photo Access：測試照片權限，未來用於旅程照片匹配。
  • Request Notifications：測試通知權限，未來用於飛行事件提醒。

實際飛行時的建議順序是：先在 iOS 主畫面開始 recording，再開 Replay Engine 看 live 地球；飛行中不要用瀏覽器版本紀錄，因為 live GPS bridge 只存在 iOS app 內嵌 WKWebView。

測試建議

  1. 先用範例 CI100 檢查地球、白線、綠線、A380、HUD 和 timeline 是否正常。
  2. 拖動時間軸，確認綠色已飛航跡會跟著變長，ETA 和距離會更新。
  3. 切換每個 Camera 模式，確認畫面不是黑屏且可以滑動旋轉、縮放。
  4. 按 GPX、KML 匯出後，用文字編輯器確認有 track point 和 waypoint。
  5. 手機版會把地球、Travel Atlas、航班預載、旅遊紀錄與控制列排成可捲動版面。
  6. 實機測 live GPS 時，可先在地面走動或開車短距離測試：HUD 應出現 LIVE，綠色已飛航跡應只跟著真實 GPS 點延伸。
