快速開始

1. 打開回放

進入首頁後會自動載入範例航班 CI100，也可以在右側「… / 更多」打開「航班預載 / API key」只輸入航班號。若有 aviationstack API key，可填入第一列「aviationstack API key（保存在本機）」；查詢成功後會快取航班，之後 API 失敗時會先用歷史航班回補。

2. 播放或暫停

按 播放 或 暫停 控制飛行回放。時間軸滑桿可拖曳到任意進度，速度選單可切換 1x、5x、20x、100x。

3. 切換視角

右側直列按鈕可切換 追、全、中、俯、塔、駕 六種視角。

4. 匯出紀錄

按 Export 下載 Travel Globe 專用包；按 GPX 或 KML 給 GPS 工具或 Google Earth 使用。

畫面說明

HUD 卡片 顯示航班號、航線、高度、速度、地速、航向、已飛距離、總距離與 ETA。
Flight Overlay 白線代表 Flight Plan，綠線代表目前已飛航跡，黃色點代表 Takeoff、TOC、TOD、Landing 等事件。
旅遊紀錄 可點擊旅程事件直接跳到該時間點，中央紀錄卡會同步顯示地點、區域與航段資訊。
航班預載 / API key 手機版按右側「… / 更多」會直接展開此區。可輸入 aviationstack API key、航班號、起降機場、日期、時間、航程分鐘與機型。流程會先嘗試 aviationstack，再使用本機航班快取，最後 fallback 到離線 seed 或手動 IATA。
Travel Atlas 顯示旅程統計、起降機場 detail、OurAirports frequency/navaid 摘要、離線包、本機歷史旅程與通知狀態。
Below / Nearby 顯示目前下方區域、正在穿越的空域/海域，以及附近城市或地標距離。

操作控制

播放 / 暫停 開始或暫停目前 journey 的時間回放。進入 iOS Live GPS 後會顯示 LIVE，表示畫面由真實 GPS 推動，不再用模擬時間播放。
1x / 5x / 20x / 100x 回放速度選單；只影響 ReplayClock，不改動原始 GPS 點或匯出資料。
時間軸滑桿 拖曳到航程任意百分比；HUD、綠色已飛航跡、附近地標、Travel Atlas 統計都會跟著刷新。
追 / 全 / 中 / 俯 / 塔 / 駕 右側視角列。依序是追機視角、完整航線、中段飛行、俯視航線、塔台視角、駕駛艙前窗視角。進入 駕 後會收合成浮動單顆按鈕，再點一次可展開視角選單。
更多 打開或收合右側抽屜。手機版會把航班預載、Travel Atlas、旅遊紀錄與匯出按鈕收進這個抽屜。
Import 選擇並匯入 .json、.travelglobe 或 zip 旅程檔；成功後目前地球、timeline 與本機歷史旅程會改成匯入內容。
Export 匯出完整 Travel Globe package，保留 journey、事件、航線、媒體 metadata 與 app 需要的 replay 資料。
Share 匯出隱私處理後的分享版 JSON，會移除或簡化不適合公開的本機私有資訊。
使用手冊 打開本頁說明；iOS 離線 bundle 也會包含這個檔案。
GPX / KML 匯出給 GPS 工具或 Google Earth 使用的航跡格式；GPX 偏向軌跡分析，KML 會包含航線與事件點。
Journal 產生離線飛行日誌 Markdown，可作為旅程摘要或手動整理紀錄的起點。
Pack 快速把 Core Global Atlas 標記為離線可用；完整資料包狀態與刪除/標記可在 Travel Atlas 裡操作。

航班預載 / API key

aviationstack API key 可留空。輸入後保存在本機 localStorage；按 套用航線 時會先嘗試 aviationstack，成功後寫入本機航班快取。
航班號 輸入如 CI100、FD234。若離線 seed 或本機快取有資料，會自動帶入起飛、抵達、時間、航程分鐘與機型。
起飛 / 抵達 輸入 IATA、ICAO、城市或機場名稱。欄位下方會出現本機機場建議，可點選機場代碼帶入表單。
日期 / 時間 設定預載航線的起飛日期與本地起飛時間；會影響事件時間、ETA、日夜光照與回放時間軸。
航程分鐘 手動指定航程長度。留空時會使用 aviationstack、本機快取、離線 seed 或距離估算。
機型 可選自動、A320、A321、B737、B767、B777、B787、A350、A380。自動時會使用 aviationstack、快取、seed 或 OpenFlights equipment fallback。
套用航線 建立 planned journey，刷新地球航線、HUD、timeline、Travel Atlas，並在 iOS WebView 內把 flight plan 送回 native 作為後續 GPS recording 的目標。

Travel Atlas

旅程統計 顯示 Trips、Countries、Years、Distance、Plan、Journal、Offline、Data、Flight context、Recording、Notice 等摘要。
機場 detail 航線起點與終點會顯示機場名稱、城市、跑道數、常用頻率與附近導航台。這些資料來自本機 OurAirports aviation context index。
機場資料庫搜尋框 這是離線查詢工具，不是航班建立表單。可搜尋 IATA、ICAO、城市或國家，用來檢查機場、跑道、頻率、導航台與 route graph 摘要；點列表任一機場會選取該代碼。
設為起飛 / 設為抵達 把機場資料庫選到的代碼帶入「航班預載 / API key」的起飛或抵達欄位，接著確認日期、時間、航班號與機型，再按 套用航線 建立 flight plan。
只看定期航班 機場資料庫的 checkbox。勾選時只列 scheduled service 機場；取消勾選時會包含較小或非定期航班機場。
Core Global Atlas 地圖、國界、城市、機場、跑道、頻率與導航台資料包。它支援 Travel Atlas、機場資料庫搜尋、起降機場 detail 與離線地理顯示。
FlightGear Global Airway Graph 全球航路、航路點與 airway graph 資料包。它支援航班預載時用真實航路連接起降機場；找不到可連接路徑時才退回 Great Circle estimate。
已內建 代表資料已隨目前 Replay build 或 iOS bundle 一起提供。FlightGear data 已納入本機包，因此不需要「使用離線」或「取消離線」；離線時會直接使用這些資料。
本機歷史旅程 載入 / 刪除 載入 會切換目前地球到該 journey；刪除 會先顯示 確認刪除 / 取消，確認後只移除 localStorage 裡的本機歷史，不影響已匯出的檔案。目前正在播放的 journey 會顯示 使用中，需先載入其他旅程再刪。
通知 Replay Engine 會把目前 notification rules 送到 iOS native local notification；瀏覽器模式只會顯示提醒狀態。

視角模式

  • 追：追機視角，鏡頭跟著目前位置與航向移動，適合觀察實際航跡。
  • 全：完整航線，拉遠看起飛、降落與整條 planned/actual route。
  • 中：中段飛行，快速切到巡航附近的構圖。
  • 俯：俯視航線，適合判斷偏航、航路點與機場接近狀態。
  • 塔：塔台視角，靠近機場起降段時會看見較強的地面焦點。
  • 駕：駕駛艙前窗視角，不顯示完整 3D 地球；畫面保留擋風玻璃框、上方遮光/ceiling、下方儀表台上緣、平直地平線、雲層與遠方地表。HUD 只顯示估算 IAS、ALT、pitch ladder、bank、HDG 與 VS；HUD 按鈕可關閉，關閉後仍能看出是駕駛艙。

匯出資料

  • .travelglobe：App 自己的完整回放格式，保留旅程、航線、事件與設定。
  • .json：適合開發、檢查 schema 或分享給其他工具處理。
  • .gpx：可匯入 GPS 工具、運動紀錄工具或航跡分析工具。
  • .kml：可匯入 Google Earth，會包含白色 Flight Plan、綠色 Actual Track 與事件點。

離線使用

Replay Engine 目前的核心回放不依賴外部網路；Blue Marble、Earth lights/cloud/specular 地球貼圖、Natural Earth 海岸線/國界/城市/地理區域、GeoNames 全球城市、OurAirports 機場/頻率/導航台索引、FlightGear 全球航路圖、授權飛機模型庫和範例資料都包在本地 build 裡。飛行前可先打開網頁或 iOS app 確認載入，飛行中即使沒有網路仍可回放已內建或已匯入的旅程。

「航班預載」會優先使用使用者本機保存的 aviationstack API key 查詢航班；成功後會寫入本機航班快取。若 API 失敗、網路不可用或沒有 key，會依序使用本機歷史航班、離線班表 seed 與手動 IATA 欄位建立 planned route、事件、ETA 和旅遊紀錄。航線會優先使用 FlightGear GPL v2 全球 airway graph 連接起降機場附近航路點；找不到可連接路徑時才退回 Great Circle estimate。OpenFlights routes.dat 只在 aviationstack 或使用者沒有提供機型時，用相同起降機場的 equipment code 補機型。

離線資料自動更新

Web 版的離線資料跟著 GitHub Pages 部署更新。iOS app 也會在啟動或回到前景時檢查 https://alien0077.github.io/Travel_Globe/ 上的 Replay Engine 與 offline-packs/core-global manifest；若遠端資料比本機新，會在有網路時下載到 app sandbox，驗證 SHA256 後切換成新版。

iOS 下載內容包含 index.html、index.js、index.css、readme.html，以及 Core Global Atlas 的 geo/airport/aviation context JSON、FlightGear 全球航路圖與 GPL v2 授權聲明。下載失敗或校驗失敗時會保留 app bundle 內建版本，不會破壞原本可離線使用的 Replay Engine。

若想立刻檢查，可在 iOS 主畫面 Replay 卡片按 更新離線資料。更新完成後重新打開 Replay Engine，即會優先使用下載版；沒有網路時仍使用上次成功下載版或 app 內建版。

Live GPS 飛行模式

iOS app 可用 iPhone CoreLocation 接收真實 GPS，透過 native bridge 即時送進 Replay Engine。此模式不需要行動網路；飛航模式下 GPS 仍可能可用，但定位品質會受座位、窗邊距離、機身遮蔽與首次定位時間影響。

  1. 起飛前先開 Travel Globe。若有網路，可先在主畫面按 更新離線資料，讓 app 抓 GitHub Pages 上最新的 Replay Engine 與離線 pack。
  2. 按 Open Replay Engine，手機版再按右側「… / 更多」，在「航班預載 / API key」輸入 aviationstack API key（可留空）、航班、起飛/抵達、日期、時間與機型，按 套用航線。可重複套用多個 flight plan；此時只建立待飛行航線，不會建立完成的旅遊紀錄。
  3. 按左上返回回到 iOS 主畫面，在 Record Into 選單挑要錄入的行程，再按 Start。iOS 會把 GPS recording 綁到被選到航線的 segmentId；若沒有套用航線，則建立 GPS-only recording。
  4. 按 Open Replay Engine 回到地球畫面。收到第一筆 GPS 後，控制列會進入 LIVE，HUD 會顯示 Live GPS 或 GPS 狀態。
  5. 若需要開飛航模式，開啟後把手機放在靠窗位置，等待 GPS 重新定位。定位恢復後飛機會繼續沿真實位置移動。
  6. 短暫斷訊時，畫面會用最後可信速度與航向最多推算 90 秒，HUD 會顯示 GPS 推算；超過 90 秒會停止外推，飛機停在最後可信位置並顯示 GPS signal lost。
  7. 落地或不需要紀錄時回到 iOS 主畫面按 Stop，旅程會保存成真實 GPS 航跡，並把 completed journey 回寫到 Replay Engine 的旅遊紀錄與 Travel Atlas。

資料庫只保存品質合格且有意義變化的真實 GPS 點；畫面推算點只留在記憶體，不會寫入 SQLite，也不會污染 GPX/KML 或之後的飛行紀錄。飛行 profile 會丟棄座標無效、水平精度過差或時間倒退的點，並以至少 5 秒、超過 250 公尺、航向或高度明顯變化作為保存門檻。

如果只是想在某個小鎮、車站、景點留下「到此一遊」，不用開始整段 GPS recording。回到 iOS 主畫面按 GPS打卡 會抓一次目前 CoreLocation 並新增到目前行程；按 照片打卡 會從該行程時間範圍內含 GPS 的照片匯入打卡點。兩者都寫入 visit_points，不會改動原始飛行 GPS 航跡。

旅遊紀錄流程

  1. 套用航線：只建立 planned flight plan，包含航班號、起降機場、機型、預估航線與 segmentId。可先建立多筆；這一步不算完成旅遊紀錄。
  2. 選擇行程並開始 GPS：按 iOS 主畫面的 Record Into 選單挑目標 flight plan，再按 Start。native recording 會使用被選到的 flight plan；每個真實 GPS 點都會保存同一個 segmentId，用來對回該航段。
  3. 停止 GPS：按 Stop 後，app 才把 recording 標記為 completed，產生實際 GPS 航跡、起訖事件、旅遊紀錄與 Travel Atlas 摘要。
  4. GPS打卡 / 照片打卡：在 iOS 主畫面可把目前 GPS 或照片 EXIF GPS 加到目前行程。Replay Engine 收到 native bridge 後會顯示成「GPS打卡」或「照片打卡」旅遊紀錄。
  5. 人工編輯：在「旅遊紀錄」面板可新增事件、修改目前紀錄、調整分類/時間、附加本機照片縮圖、復原上次編輯、隱藏紀錄，或編輯航線摘要。人工修改只寫入 edit overlay、media metadata 或 journey metadata，不會改掉原始 GPS 點。
  6. 資料對應：GPS record 是原始軌跡；Visit Point 是同一行程下的到此一遊點；Travel Record 是同一個 segmentId 下整理出的旅遊事件。刪除/隱藏旅遊事件不會刪除 SQLite 裡的真實 GPS。

旅遊紀錄面板控制項

區域篩選按鈕 All 與各區域按鈕會篩選 timeline 列表；切到特定區域時會自動選中該區域第一筆紀錄。
Timeline 紀錄列 點任一旅遊紀錄會把時間軸跳到該事件時間，並同步更新中央紀錄卡。
新增 / 新增事件 在紀錄卡內開啟新增表單，用目前地球顯示位置建立人工打卡事件；輸入標題與備註後按 新增。
修改 / 修改紀錄 在紀錄卡內開啟修改表單，更新目前紀錄標題與副標題，寫入 edit overlay，不改原始 GPS。
分類/時間 在紀錄卡內開啟分類與時間表單，可選區域並用日期/時間欄位調整事件時間；無效輸入會保留原值。
附加照片 選取本機照片並附加到目前紀錄。預設標記為本機私有，匯出分享版時會依隱私規則處理。
載入最新 iOS WebView 內要求 native 從 SQLite 載入最新 journey；瀏覽器模式會顯示沒有 iOS SQLite 的狀態訊息。
復原 / 復原上次 復原最近一次旅遊紀錄編輯，包含新增、修改、分類、隱藏、附加照片與航線摘要編輯。最多保留最近數筆編輯堆疊。
隱藏/刪除 / 隱藏紀錄 在紀錄卡內顯示確認訊息，確認後把目前旅遊事件標記為 hidden；原始 GPS、segment 與 SQLite 紀錄仍保留，可用 復原 取回。
編輯航線 / 編輯航線摘要 在紀錄卡內開啟航線摘要表單，可修改航班號、機型、起飛 IATA、抵達 IATA 與 journey 標題摘要。

iOS App 使用

  • Start：在 Recorder 卡片開始紀錄真實 GPS 航跡；這是真實飛行的錄製入口。
  • Stop：停止並保存旅程。
  • Record Into：選擇這次 GPS recording 要寫入哪一筆已預載 flight plan；若沒有選到合適航線，會建立 GPS-only recording。
  • Open Replay Engine：打開內嵌 Web Replay Engine；用來預載航線、看 live 地球、看回放與匯出。
  • GPS打卡：用目前 iPhone GPS 在目前行程新增一筆 visit point。
  • 照片打卡：從行程時間範圍內有 GPS 的 PhotoKit 照片匯入 visit points。
  • 載入最新紀錄：把 SQLite 最新 journey queue 回 Replay Engine，適合 app 重新打開後繼續看剛錄完的旅程。
  • 更新離線資料：有網路時檢查 GitHub Pages 上由 GitHub Actions 發布的新 Replay Engine 與 offline pack，下載成功後下次打開 Replay Engine 會使用新版資料。
  • Photos：要求或更新 PhotoKit 權限，用於照片 GPS 匯入。
  • Notifications：要求或更新通知權限；Replay Engine notification rules 會透過 native bridge 排程 local notification。

實際飛行時的建議順序是：先在 Replay Engine 套用航線，再回 iOS 主畫面選 Record Into 並按 Start，最後回 Replay Engine 看 live 地球；飛行中不要用瀏覽器版本紀錄，因為 live GPS bridge 只存在 iOS app 內嵌 WKWebView。

測試建議

  1. 先用範例 CI100 檢查地球、白線、綠線、A380、HUD 和 timeline 是否正常。
  2. 拖動時間軸，確認綠色已飛航跡會跟著變長，ETA 和距離會更新。
  3. 切換每個 Camera 模式，確認畫面不是黑屏且可以滑動旋轉、縮放。
  4. 按 GPX、KML 匯出後，用文字編輯器確認有 track point 和 waypoint。
  5. 手機版會把地球、Travel Atlas、航班預載、旅遊紀錄與控制列排成可捲動版面。
  6. 實機測 live GPS 時，可先在地面走動或開車短距離測試：HUD 應出現 LIVE，綠色已飛航跡應只跟著真實 GPS 點延伸。
